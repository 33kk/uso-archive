### Description
<br><b>YOU NEED TO USE 'Advanced Style Settings' BUTTON DUE TO NEW SITE DESIGN CHANGES !!!</b><br><br>Allows you to reveal 'List all tabs' toolbar (tab bar) button, and customize it with tabs counter and menu items numbering, among others. Also to style 'New tab' button as regular button instead of tab button, or to hide it.

### Update notes
To make this userstyle I have taken solutions from
<a href="https://userstyles.org/styles/85286/firefox-display-total-number-of-tabs">https://userstyles.org/styles/85286/firefox-display-total-number-of-tabs</a>

and a proposal from <a href="https://userstyles.org/users/2247">haleakari</a>, together with my own features and fixes.

This userstyle allows you to reveal 'List all tabs' menu button (if not showed in toolbars nor through toolbars 'Customize' dialog), hides 'List all tabs' menu faulty pseudo-scrollbar, and lets you choose some features:
<ul><li>Revealing list all tabs button always, or only on tab overflow.</li>
<li>Adding tabs counter to 'List all tabs' button, with custom color. By default it does count pinned and hidden tabs (hidden with HideTab addon), but you can choose not to count each one of these types, or both. Counter is only revealeed with 'List all tabs' button placed in a toolbar where counter actually works (due to DOM restrictions). In some old Firefox versions (Firefox standard is 52 at the time I'm writting this) counter is only revealed in the tab bar or the addon bar. In modern Firefox versions counter is not revealing only for menu bar. Additionally counter is revealed for toolbars generated by Classic Theme Restorer addon. Also I could verify at least for Firefox standard 46.0.1 (when I checked this) 'List all tabs' button couldn't be moved between toolbars by default, if this is your case that can be changed as explained <a href="https://forum.userstyles.org/discussion/49992/works-great">here</a>.</li>
<li>Hiding drop down arrow from 'List all tabs' button.</li>
<li>Adding numbering to 'List all tabs' menu items. This menu shows hidden tabs but not pinned tabs, so if later are used but they are not excluded by setting from the button counter, or if hidden tabs are used but they are excluded by setting from the counter, total items number can be different from button counter value.</li>
<li>Hiding 'Tab Groups' (panorama) item from 'List all tabs' menu (if available on the browser, it's still accesible via toolbar button or through keyboard shortcut: Ctrl+Shift+E, note this feature has been removed on modern Firefox versions and now is offered by some addons I have not tested).</li>
<li>Hiding 'Undo Last Closed Tab' item from 'List all tabs' menu (still accesible via toolbar button or through keyboard shortcut: Ctrl+Shift+T).</li>
<li>Hiding 'New Container Tab' submenu from 'List all tabs' menu (still accesible via File menu, note Classic Theme Restorer addon has an option to disable this feature completely).</li>
<li>Increasing 'List all tabs' menu maximum width.</li>
<li>Changing 'New tab' button style not to show it as a tab (if applicable), but as a regular button aligned with the rest of the tab bar buttons <b>when there's no tabs overflow and it is the first item after the tabs</b> (otherwise it usually acts as regular button too, see 'Without this style' screenshot).</li>
<li>Hiding 'New tab' button.</li>
</ul>
It's probably a good idea to disable tabs animation to avoid glitches when tabs number grows to the tab bar overflow triggering limit: Overflow can happen during tabs animation (e.g. on tab opening) despite after animation finishes no overflow remains, in that specific case the emerging overflow-style elements (e.g. tabs scrollbuttons or 'List all tabs' button, where applicable) arise recurrently triggering a new overflow, causing a loop of continuous flashing overflow-style elements. To avoid this <b><i>browser.tabs.animate</i></b> under <b><i>about:config</i></b> should be set to false.

I've tested this style up to FF 55.0a1. In the screenshots I'm also using Classic Theme Restorer and Hide Caption Titlebar Plus addons.

Please don't hesitate to comment any problems you find, I might not be aware of new Firefox changes, or have not detected some needing or some issue on some version. About DOM restrictions not allowing 'List all tabs' button counter to work (zeroed) on some toolbars, as described above: tab bar elements are out of the scope of the counter variable on that DOM tree positions; if you have a CSS solution for this issue I'll be glad to include it.
